## 1. Introduccion

### 1.1 Proposito del Documento

Este documento establece el estandar oficial para la creacion de paginas de servicios dentro del sitio web de GUPRI Seguridad Privada. Su objetivo es garantizar que cada nueva pagina de servicio:

- Mantenga consistencia visual, tecnica y semantica con el resto del sitio
- Maximice el posicionamiento organico en motores de busqueda
- Optimice la tasa de conversion de visitantes a leads calificados
- Facilite la escalabilidad del sitio sin perder autoridad SEO
- Fortalezca la arquitectura de enlaces internos

### 1.2 Importancia de la Homologacion de Servicios

La homologacion garantiza que:

- Google interprete correctamente la jerarquia y relacion entre servicios
- Los usuarios tengan una experiencia predecible y profesional
- El equipo pueda crear nuevas paginas de forma eficiente
- Se evite la canibalizacion de palabras clave entre paginas
- Se mantenga la autoridad de dominio distribuida estrategicamente

### 1.3 Beneficios SEO y de Marketing

| Beneficio | Impacto |
|-----------|---------|
| Consistencia semantica | Mejora el crawling de Google |
| Estructura de enlaces | Distribuye autoridad entre paginas |
| Schema markup estandarizado | Habilita rich snippets |
| CTAs optimizados | Aumenta tasa de conversion |
| Contenido especializado | Posiciona long-tail keywords |

## 2. Investigacion Previa del Servicio

Antes de crear cualquier pagina de servicio, completa esta investigacion:

### 2.1 Analisis de Intencion de Busqueda

Identifica el tipo de intencion detras de las busquedas relacionadas con el servicio:

| Tipo de Intencion | Ejemplo | Enfoque de Contenido |
|-------------------|---------|----------------------|
| Transaccional | "contratar guardias para condominio" | CTA directo, formulario, cotizacion |
| Informativa | "como funciona la seguridad privada" | Contenido educativo, FAQ |
| Comparativa | "mejor empresa de seguridad CDMX" | Diferenciadores, testimonios |
| Local | "seguridad privada cerca de mi" | Cobertura, ubicaciones |

**Proceso:**
1. Busca la keyword principal en Google
2. Analiza los primeros 10 resultados organicos
3. Identifica el patron de contenido dominante
4. Adapta tu contenido para satisfacer esa intencion

### 2.2 Validacion de Palabra Clave Principal

**Criterios de seleccion:**

- Volumen de busqueda mensual estimado (minimo 100 busquedas/mes)
- Dificultad de posicionamiento (preferir media-baja para nuevas paginas)
- Relevancia comercial para GUPRI
- Capacidad de diferenciacion frente a competidores

**Herramientas recomendadas:**
- Google Keyword Planner
- Google Trends
- Autocompletado de Google
- Busquedas relacionadas al final de SERP

**Formato de keyword principal:**
```
[servicio] + [modificador geografico/calificador]

Ejemplos:
- "seguridad para condominios"
- "guardias para empresas mexico"
- "vigilancia industrial cdmx"
- "escoltas ejecutivos monterrey"
```

### 2.3 Seleccion de Palabras Clave Secundarias y Long-tail

Para cada servicio, identifica:

| Tipo | Cantidad | Ejemplo |
|------|----------|---------|
| Keyword principal | 1 | seguridad para condominios |
| Keywords secundarias | 3-5 | guardias para condominios, vigilancia condominios, seguridad residencial |
| Long-tail | 5-10 | contratar seguridad para condominio en cdmx, empresa de vigilancia para fraccionamientos |
| Keywords LSI | 5-8 | control de acceso, rondines, CCTV, intramuros, certificacion REPSE |

### 2.4 Analisis Rapido de Competencia

Antes de crear el contenido:

1. **Busca la keyword principal en Google**
2. **Analiza los 5 primeros resultados:**
   - Longitud del contenido (palabras)
   - Estructura de headings (H1, H2, H3)
   - Tipo de contenido (landing, articulo, directorio)
   - Elementos multimedia
   - Schema markup utilizado

3. **Identifica gaps de contenido:**
   - Temas que no cubren
   - Preguntas sin responder
   - Aspectos locales no mencionados

4. **Define tu diferenciador:**
   - Que ofreces que ellos no
   - Enfoque mas local/especializado
   - Pruebas sociales mas fuertes

### 2.5 Definicion del Enfoque Comercial

Antes de escribir, define claramente:

```
SERVICIO: [Nombre del servicio]
KEYWORD PRINCIPAL: [keyword target]
CLIENTE IDEAL: [perfil del decision maker]
PROBLEMA PRINCIPAL: [pain point que resolvemos]
SOLUCION GUPRI: [como lo resolvemos]
DIFERENCIADOR CLAVE: [por que elegirnos]
CTA PRINCIPAL: [accion deseada]
```

## 3. Convenciones de Naming y URLs

### 3.1 Estructura de URL

**Formato obligatorio:**
```
https://guardias-privados.com/[servicio-palabra-clave].html
```

**Reglas:**
- Solo minusculas
- Palabras separadas por guiones medios (-)
- Sin articulos (el, la, los, de, para) a menos que sean parte integral de la keyword
- Maximo 60 caracteres
- Sin numeros a menos que sean relevantes
- Sin caracteres especiales ni acentos

**Ejemplos correctos:**
```
seguridad-para-condominios.html
guardias-industriales.html
escoltas-ejecutivos.html
vigilancia-empresarial.html
seguridad-eventos-corporativos.html
```

**Ejemplos incorrectos:**
```
seguridad_condominios.html       (guion bajo)
SeguridadCondominios.html        (mayusculas)
seguridad-para-los-condominios.html (articulo innecesario)
servicio-1.html                  (no descriptivo)
```

### 3.2 Reglas para Evitar Canibalizacion SEO

| Situacion | Solucion |
|-----------|----------|
| Dos servicios con keywords similares | Diferenciar con modificador geografico o de especializacion |
| Servicio general vs especifico | El especifico enlaza al general como categoria padre |
| Misma keyword en blog y servicio | El servicio es la pagina principal, el blog enlaza a el |

**Matriz de diferenciacion:**
```
Servicio general:    /seguridad-empresarial.html
Servicio especifico: /seguridad-para-bancos.html
                     /seguridad-para-corporativos.html
                     /seguridad-para-comercios.html
```

### 3.3 Nomenclatura de Archivos Relacionados

| Tipo de archivo | Formato | Ejemplo |
|-----------------|---------|---------|
| Pagina HTML | servicio-keyword.html | seguridad-para-condominios.html |
| Imagen OG | og-servicio-keyword.jpg | og-seguridad-condominios.jpg |
| Imagen hero | hero-servicio.jpg | hero-seguridad-condominios.jpg |
| Imagenes internas | servicio-descripcion.jpg | guardias-condominios-acceso.jpg |

## 4. Estructura SEO del Contenido

### 4.1 Arquitectura de la Pagina

Basado en la plantilla `/seguridad-para-condominios.html`, cada pagina de servicio debe contener estas secciones en orden:

```
1. HEAD (Meta tags, Schema, OG)
2. Top Bar (contacto)
3. Header (navegacion con dropdown)
4. MAIN
   4.1 Hero Section (breadcrumb, H1, CTA)
   4.2 Service Overview (descripcion principal)
   4.3 Benefits Section (beneficios, diferenciadores)
   4.4 Process Section (como funciona)
   4.5 Proposal Section (planes/propuesta)
   4.6 Testimonials (prueba social)
   4.7 FAQ Section (preguntas frecuentes)
   4.8 Contact/Quote Section (formulario)
   4.9 Related Services CTA (servicios relacionados)
5. Footer
6. WhatsApp Float
```

### 4.2 Estructura de Headings

**H1 - Titulo Principal (unico por pagina)**

Formato:
```
[Keyword Principal]: [Beneficio/Propuesta de Valor] | [Diferenciador]
```

Ejemplo:
```html
<h1>Seguridad para Condominios: Proteccion Integral 24/7 con Guardias Certificados</h1>
```

Requisitos:
- Maximo 70 caracteres
- Incluye keyword principal al inicio
- Orientado a conversion (beneficio tangible)
- Unico en todo el sitio

**H2 - Secciones Principales**

Cada seccion debe tener un H2 estrategico:

| Seccion | Formato H2 | Ejemplo |
|---------|------------|---------|
| Overview | Keyword + Especificacion | "Seguridad Integral para Condominios Verticales y Horizontales" |
| Beneficios | Ventajas de + Servicio | "Ventajas de Contratar Seguridad Profesional para tu Condominio" |
| Proceso | Como Funciona + Servicio | "Como Funciona Nuestro Servicio de Seguridad para Condominios" |
| Propuesta | Planes/Opciones para + Target | "Planes Flexibles para tu Condominio" |
| Testimonios | Lo que Dicen + Tipo de Cliente | "Lo que Dicen los Administradores de Condominios" |
| FAQ | Todo lo que Necesitas Saber sobre + Servicio | "Todo lo que Necesitas Saber sobre Seguridad para Condominios" |
| Contacto | Accion + Beneficio | "Solicita tu Cotizacion Gratuita" |
| Servicios CTA | Servicios Complementarios | "Servicios Complementarios para tu Condominio" |

**H3 - Subsecciones de Soporte**

Usar para:
- Beneficios individuales dentro de la seccion de beneficios
- Pasos del proceso
- Titulos de tarjetas de servicio
- Preguntas en FAQ (dentro de `<summary>`)

**H4 - Elementos Menores**

Usar para:
- "El servicio incluye:" (listas de features)
- "Lo que Incluye Nuestra Propuesta:"
- Subtitulos dentro de cards

### 4.3 Jerarquia Semantica Correcta

```
<h1> Titulo principal de la pagina (1)
  <h2> Seccion Overview (1)
  <h2> Seccion Beneficios (1)
    <h4> Lista de features
  <h2> Seccion Proceso (1)
    <h3> Paso 1
    <h3> Paso 2
    ...
  <h2> Seccion Propuesta (1)
    <h4> Lo que incluye
  <h2> Seccion Testimonios (1)
  <h2> Seccion FAQ (1)
  <h2> Seccion Contacto (1)
    <h3> Titulo del formulario
  <h2> Servicios Relacionados (1)
```

### 4.4 Longitud del Contenido

| Seccion | Palabras Minimas | Palabras Optimas |
|---------|------------------|------------------|
| Hero subtitle | 40 | 60 |
| Service Overview | 200 | 350 |
| Beneficios (total) | 300 | 500 |
| Proceso | 150 | 250 |
| Propuesta | 150 | 250 |
| FAQ (total) | 400 | 600 |
| **TOTAL PAGINA** | **1,500** | **2,500** |

### 4.5 Densidad de Keywords

| Elemento | Keyword Principal | Keywords Secundarias |
|----------|-------------------|----------------------|
| H1 | 1 vez (obligatorio) | 0 |
| H2 | 2-3 veces (distribuido) | 1-2 veces |
| Primer parrafo | 1 vez | 1-2 veces |
| Body content | 4-6 veces natural | 3-5 veces |
| Alt images | 1-2 veces | 1-2 veces |
| **Densidad total** | **1-2%** | **0.5-1%** |

## 5. Elementos de Marketing Obligatorios

### 5.1 Propuesta de Valor Clara

Ubicacion: Hero section y Marketing Card

Formato:
```
Somos [posicionamiento] en [servicio] en [ubicacion].
[Beneficio cuantificable] + [Diferenciador unico].
```

Ejemplo:
```
Somos la empresa lider en seguridad privada para condominios en Mexico.
Nuestro modelo de operacion ha demostrado una reduccion del 85% en incidentes
durante el primer ano de servicio.
```

### 5.2 Diferenciadores Frente a la Competencia

Incluir minimo 5 diferenciadores, presentados como beneficios:

| Diferenciador GUPRI | Formato de Presentacion |
|---------------------|-------------------------|
| 30+ anos experiencia | "Mas de 30 anos como empresa lider" |
| Certificacion REPSE | "100% cumplimiento legal - Registro REPSE" |
| Cobertura geografica | "Operamos en CDMX, Edomex, GDL, MTY, QRO" |
| Personal certificado | "Solo el 15% de candidatos pasa nuestro proceso" |
| Respuesta rapida | "Supervision en sitio en menos de 15 minutos" |
| Transparencia | "Sin cargos ocultos ni letra pequena" |

### 5.3 Beneficios Tangibles para el Cliente

Cada beneficio debe responder: "Que gano yo?"

| Feature | Beneficio Tangible |
|---------|-------------------|
| Guardias certificados | "Tranquilidad de saber que tu familia esta protegida por profesionales" |
| Control de acceso | "Sabes exactamente quien entra y sale de tu comunidad" |
| Rondines GPS | "Visibilidad total de donde estan los guardias en tiempo real" |
| Respuesta rapida | "Ante cualquier emergencia, apoyo en menos de 15 minutos" |

### 5.4 Objeciones Comunes y Como Resolverlas

Incluir en FAQ y/o contenido:

| Objecion | Respuesta en Contenido |
|----------|------------------------|
| "Es muy caro" | Mostrar valor incluido: uniformes, equipo, seguros, supervision |
| "No se si cumplen la ley" | Destacar REPSE, IMSS, SSP, STPS |
| "Que pasa si falta un guardia" | Explicar cobertura 100% con personal de relevo |
| "No conozco la empresa" | Testimonios, casos de exito, numeros de clientes |
| "No se si opera en mi zona" | Seccion clara de cobertura |

### 5.5 CTAs Estrategicos

**CTA Principal (Hero):**
```html
<a href="#cotizar" class="btn btn-primary btn-lg">
  Cotizar [Servicio] para [Target]
  <svg>...</svg>
</a>
```

**CTA Secundario (WhatsApp):**
```html
<a href="https://wa.me/521TELEFONO?text=Mensaje" class="btn btn-whatsapp btn-lg">
  WhatsApp Directo
</a>
```

**Ubicacion de CTAs:**
1. Hero section (2 CTAs: principal + WhatsApp)
2. Final de Service Overview
3. Final de Propuesta section
4. Seccion de Contacto completa
5. WhatsApp flotante (fijo)
6. Header (boton Cotizar)

### 5.6 Pruebas de Confianza (Trust Signals)

Incluir obligatoriamente:

**En Hero (trust-items):**
- Numero de clientes/condominios protegidos
- Certificacion REPSE
- Anos de experiencia

**En Beneficios:**
- Estadisticas de reduccion de incidentes
- Porcentaje de renovacion de contratos
- Calificacion promedio de clientes

**En Footer:**
- Logos de certificaciones: REPSE, SSP, IMSS, STPS

## 6. SEO On Page Avanzado

### 6.1 Meta Title

**Formato:**
```
[Keyword Principal] en [Ubicacion] | [Beneficio/CTA] | GUPRI
```

**Requisitos:**
- 50-60 caracteres (maximo 60)
- Keyword principal al inicio
- Incluir marca al final
- Separadores: | o -

**Ejemplo:**
```html
<title>Seguridad para Condominios en Mexico | Guardias 24/7 | GUPRI</title>
```

### 6.2 Meta Description

**Formato:**
```
[Propuesta de valor]. [Beneficios clave]. [Prueba social]. [CTA].
```

**Requisitos:**
- 150-160 caracteres (maximo 160)
- Keyword principal en primeras palabras
- Incluir call-to-action
- Usar numeros cuando sea posible

**Ejemplo:**
```html
<meta name="description" content="Servicio especializado de seguridad para condominios en Mexico. Guardias intramuros certificados, control de acceso 24/7, vigilancia profesional. Mas de 500 condominios protegidos. Cotiza gratis.">
```

### 6.3 Meta Keywords

```html
<meta name="keywords" content="[keyword principal], [keyword secundaria 1], [keyword secundaria 2], [long-tail 1], [long-tail 2], [LSI 1], [LSI 2]">
```

Incluir 8-12 keywords separadas por comas.

### 6.4 Open Graph Tags

```html
<!-- Open Graph / Facebook -->
<meta property="og:type" content="website">
<meta property="og:url" content="https://guardias-privados.com/[servicio].html">
<meta property="og:title" content="[Titulo optimizado para social - 60 chars]">
<meta property="og:description" content="[Descripcion para social - 200 chars]">
<meta property="og:image" content="https://guardias-privados.com/img/og-[servicio].jpg">
<meta property="og:image:width" content="1200">
<meta property="og:image:height" content="630">
<meta property="og:image:alt" content="[Descripcion de imagen]">
<meta property="og:locale" content="es_MX">
<meta property="og:site_name" content="GUPRI Seguridad Privada">
```

### 6.5 Twitter Cards

```html
<meta name="twitter:card" content="summary_large_image">
<meta name="twitter:url" content="https://guardias-privados.com/[servicio].html">
<meta name="twitter:title" content="[Titulo - 70 chars]">
<meta name="twitter:description" content="[Descripcion - 200 chars]">
<meta name="twitter:image" content="https://guardias-privados.com/img/og-[servicio].jpg">
```

## 7. Interlinking Obligatorio

### 7.1 Menu Principal

**Ubicacion:** Header > nav-dropdown-menu

**Archivo a modificar:** Todos los HTML (index.html, servicios.html, y todas las paginas de servicio)

**Codigo a agregar:**
```html
<li class="nav-dropdown">
  <a href="servicios.html" class="nav-link nav-dropdown-toggle">
    Servicios
    <svg>...</svg>
  </a>
  <ul class="nav-dropdown-menu">
    <li><a href="servicios.html" class="nav-dropdown-item">Todos los Servicios</a></li>
    <li><a href="seguridad-para-condominios.html" class="nav-dropdown-item">Seguridad para Condominios</a></li>
    <!-- AGREGAR NUEVO SERVICIO AQUI -->
    <li><a href="[nuevo-servicio].html" class="nav-dropdown-item">[Nombre Servicio]</a></li>
    <!-- FIN NUEVO SERVICIO -->
    <li><a href="servicios.html#seguridad-corporativa" class="nav-dropdown-item">Seguridad Corporativa</a></li>
    ...
  </ul>
</li>
```

**Reglas:**
- Los servicios con pagina dedicada van primero
- Ordenar por importancia comercial
- Maximo 8 items en el dropdown
- El servicio activo debe tener clase `active`

### 7.2 Footer

**Ubicacion:** Footer > footer-links (seccion Servicios)

**Archivos a modificar:** Todos los HTML

**Codigo a agregar:**
```html
<div class="footer-links">
  <h4>Servicios</h4>
  <ul>
    <li><a href="seguridad-para-condominios.html">Seguridad para Condominios</a></li>
    <!-- AGREGAR NUEVO SERVICIO -->
    <li><a href="[nuevo-servicio].html">[Nombre Servicio]</a></li>
    <!-- FIN NUEVO SERVICIO -->
    <li><a href="servicios.html#guardias-residenciales">Guardias Residenciales</a></li>
    <li><a href="servicios.html#control-acceso">Control de Acceso</a></li>
  </ul>
</div>
```

**Reglas:**
- Maximo 5-6 servicios en footer
- Solo los servicios principales/con pagina dedicada
- Mantener consistencia en todos los archivos
